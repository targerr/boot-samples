# Spring Boot Plugin 使用指南

## 项目概述

本项目是一个基于 Spring Boot 的插件集合，提供了多个实用的功能模块，可以方便地集成到您的 Spring Boot 项目中。

## 技术栈

- **Spring Boot**: 2.7.6
- **Java**: 11
- **MyBatis-Plus**: 3.4.3.4
- **Druid**: 数据库连接池

---

## 一、插件模块介绍

### 1.1 plugin-log - 日志插件

用于记录方法执行时间的简单日志插件。

#### 功能特性

- 自动记录方法执行时间
- 输出请求URI、方法名、方法描述等信息
- 基于注解的便捷使用方式

#### 使用方法

**1. 添加依赖**

```xml
<dependency>
    <groupId>com.example</groupId>
    <artifactId>plugin-log</artifactId>
    <version>1.0-SNAPSHOT</version>
</dependency>
```

**2. 在Controller方法上添加@MyLog注解**

```java
@RestController
@RequestMapping("/hello")
public class HelloController {

    @MyLog(desc = "sayHello方法")
    @GetMapping("/say")
    public String sayHello() {
        return "Hello World";
    }
}
```

**3. 控制台输出示例**

```
请求uri：/hello/say
请求方法名：com.example.controller.HelloController.sayHello
方法描述：sayHello方法
方法执行时间：36ms
```

#### 注解说明

| 注解 | 参数 | 说明 |
|------|------|------|
| @MyLog | desc | 方法描述，用于日志输出 |

### 1.2 plugin-echo - 字段回显插件

用于自动注入字段值的插件，可以根据配置自动将字典、关联数据等注入到对象字段中。

#### 功能特性

- 支持字典回显
- 支持关联对象回显
- 支持自定义API查询
- 支持过滤条件
- 自动缓存机制

#### 使用方法

**1. 添加依赖**

```xml
<dependency>
    <groupId>com.example</groupId>
    <artifactId>plugin-echo</artifactId>
    <version>1.0-SNAPSHOT</version>
</dependency>
```

**2. 在实体类字段上添加@EchoField注解**

```java
public class User {

    private Long userId;

    @EchoField(
        api = "dictionaryServiceImpl",
        bindKey = "nation",
        ref = "nationName"
    )
    private String nation;

    @EchoField(
        api = "userApi",
        bindKey = "userId",
        ref = "userName"
    )
    private String userName;
}
```

#### 注解说明

| 参数 | 类型 | 说明 | 示例 |
|------|------|------|------|
| ref | String | 回显到哪个字段 | "nationName" |
| bindKey | String | 绑定查询的key（根据对象属性查询） | "nation" |
| filter | String | 分类过滤的key（用于字典、配置等） | "USER_TYPE" |
| api | String | 提供自动注入值的查询类的Spring Name | "dictionaryServiceImpl" |
| beanClass | Class<?> | 自动注入值的类型，用于强制转换 | User.class |
| dictType | String | 字典类型（def_dict表的parent_key字段） | "nation_type" |

#### 使用场景示例

**场景1：字典回显**

```java
// 用户实体类
public class User {
    private Integer nation;  // 民族编码

    // 自动注入民族名称
    @EchoField(
        api = "dictionaryServiceImpl",
        dictType = "NATION_TYPE",
        bindKey = "nation",
        ref = "nationName"
    )
    private String nationName;
}
```

**场景2：关联对象回显**

```java
// 订单实体类
public class Order {
    private Long userId;  // 用户ID

    // 自动注入用户名
    @EchoField(
        api = "userApi",
        bindKey = "userId",
        ref = "userName"
    )
    private String userName;
}
```

### 1.3 plugin-dynamic-datasource - 动态数据源插件

用于实现多数据源动态切换的插件，支持主从库切换、读写分离等功能。

#### 功能特性

- 支持多数据源配置
- 基于注解的数据源切换
- 支持通过HTTP Header切换数据源
- 支持Druid监控
- 线程安全的上下文管理

#### 使用方法

**1. 添加依赖**

```xml
<dependency>
    <groupId>com.example</groupId>
    <artifactId>plugin-dynamic-datasource</artifactId>
    <version>1.0-SNAPSHOT</version>
</dependency>
```

**2. 配置数据源**

```yaml
spring:
  datasource:
    type: com.alibaba.druid.pool.DruidDataSource
    driverClassName: com.mysql.cj.jdbc.Driver
    druid:
      # 主库数据源
      master:
        url: jdbc:mysql://localhost:3306/main_db
        username: root
        password: 123456
      # 从库数据源
      slave:
        enabled: true  # 从数据源开关
        url: jdbc:mysql://localhost:3306/slave_db
        username: root
        password: 123456
      # 连接池配置
      initialSize: 5
      minIdle: 10
      maxActive: 20
      maxWait: 60000
      validationQuery: SELECT 1 FROM DUAL
      testWhileIdle: true
      testOnBorrow: false
      testOnReturn: false
      # 监控配置
      statViewServlet:
        enabled: true
        url-pattern: /druid/*
        login-username: admin
        login-password: admin123
```

**3. 在Service或方法上使用@DataSource注解**

```java
@Service
public class UserServiceImpl implements UserService {

    // 使用主库
    @DataSource(DataSourceType.MASTER)
    public User getUserById(Long id) {
        return userMapper.selectById(id);
    }

    // 使用从库
    @DataSource(DataSourceType.SLAVE)
    public List<User> getUserList() {
        return userMapper.selectList(null);
    }
}
```

**4. 通过HTTP Header切换数据源**

在请求头中添加 `X-Datasource` 字段：

```bash
curl -H "X-Datasource: SLAVE" http://localhost:8080/api/users
```

#### 数据源类型

| 类型 | 说明 |
|------|------|
| MASTER | 主库（默认） |
| SLAVE | 从库 |
| GRAY | 灰度库 |

#### 注解说明

| 注解 | 参数 | 说明 |
|------|------|------|
| @DataSource | value | 数据源类型，默认为MASTER |

#### Druid监控访问

访问地址：`http://localhost:8080/druid/`

用户名和密码在配置文件中设置。

### 1.4 common-log - 通用操作日志模块

用于记录系统操作日志的通用模块，支持多种业务类型和操作类型。

#### 功能特性

- 记录操作日志（增删改查）
- 记录登录日志
- 支持请求日志记录
- 支持异步日志处理
- 支持自定义日志处理

#### 使用方法

**1. 添加依赖**

```xml
<dependency>
    <groupId>com.example</groupId>
    <artifactId>common-log</artifactId>
    <version>1.0-SNAPSHOT</version>
</dependency>
```

**2. 在方法上添加@LogOperate注解**

```java
@RestController
@RequestMapping("/api/user")
public class UserController {

    @LogOperate(value = "查询用户列表", businessType = BusinessType.SELECT)
    @GetMapping("/list")
    public List<User> list() {
        return userService.list();
    }

    @LogOperate(value = "新增用户", businessType = BusinessType.INSERT)
    @PostMapping
    public void add(@RequestBody User user) {
        userService.save(user);
    }

    @LogOperate(value = "修改用户", businessType = BusinessType.UPDATE)
    @PutMapping
    public void update(@RequestBody User user) {
        userService.updateById(user);
    }

    @LogOperate(value = "删除用户", businessType = BusinessType.DELETE)
    @DeleteMapping("/{id}")
    public void delete(@PathVariable Long id) {
        userService.removeById(id);
    }
}
```

#### 注解说明

| 参数 | 类型 | 说明 |
|------|------|------|
| value | String | 操作描述 |
| businessType | BusinessType | 业务类型（SELECT/INSERT/UPDATE/DELETE/OTHER等） |
| operatorType | OperatorType | 操作人类别（WEB/MOBILE/SYSTEM等） |
| saveRequestData | boolean | 是否保存请求参数，默认true |

#### 业务类型枚举

```java
public enum BusinessType {
    OTHER,      // 其它
    INSERT,     // 新增
    UPDATE,     // 修改
    DELETE,     // 删除
    GRANT,      // 授权
    EXPORT,     // 导出
    IMPORT,     // 导入
    FORCE,      // 强退
    CODE,       // 生成代码
    CLEAN,      // 清空数据
    SELECT      // 查询
}
```

#### 操作人类别枚举

```java
public enum OperatorType {
    WEB,        // PC端用户
    MOBILE,     // 移动端用户
    SYSTEM      // 系统内部用户
}
```

## 四、最佳实践

### 4.1 数据源切换最佳实践

1. **读写分离**：查询操作使用从库，写操作使用主库
2. **事务一致性**：在同一事务中不要切换数据源
3. **灰度发布**：使用GRAY数据源进行灰度测试

```java
@Service
public class ProductServiceImpl {

    // 查询使用从库
    @DataSource(DataSourceType.SLAVE)
    public Product getProductById(Long id) {
        return productMapper.selectById(id);
    }

    // 写入使用主库
    @DataSource(DataSourceType.MASTER)
    @Transactional
    public void saveProduct(Product product) {
        productMapper.insert(product);
    }
}
```

### 4.2 日志记录最佳实践

1. **关键操作必记日志**：新增、修改、删除操作
2. **查询操作可选**：根据业务需求决定是否记录
3. **敏感信息脱敏**：避免记录密码等敏感信息

```java
@LogOperate(value = "用户登录", businessType = BusinessType.SELECT, saveRequestData = false)
public LoginResult login(LoginRequest request) {
    // 登录逻辑，不记录请求参数（包含密码）
}
```

### 4.3 字段回显最佳实践

1. **避免循环依赖**：对象之间不要互相回显
2. **合理使用缓存**：字典等静态数据自动缓存
3. **性能考虑**：大量数据回显时注意性能

## 五、常见问题

### 5.1 数据源切换不生效

**原因**：可能是因为事务导致的数据源切换失效

**解决方案**：确保在事务外部进行数据源切换，或在类级别使用@DataSource注解

### 5.2 字段回显失败

**原因**：API配置错误或Spring Bean名称不匹配

**解决方案**：
- 检查api参数是否正确
- 确认对应的服务类存在且已注册为Spring Bean

### 5.3 日志未记录

**原因**：切面未生效或注解使用不正确

**解决方案**：
- 确认注解使用在public方法上
- 检查类是否被Spring管理（@Service或@Component）
